_Last updated: 2026-05-23_ (**Phase 11 deepens further still** (D-061): L11.3 lands a deepening of `tests/place-id.test.ts` — the canonicalisation engine that decides cache keys (D-020 `gr:reviews:v1:<slug>`), share-URL segments, and download filenames (D-033). The single-rule suite already proves each canonicalisation arrow in isolation (ChIJ recased, 0x lowercased, MOCK_ uppercased, URL extraction, whitespace trim, short-link rejection); three cross-cutting load-bearing concerns were unguarded. **First: surface-form aliasing → identical `{raw, slug}`.** This is the actual cache-key integrity contract — different surface forms of the same Place ID *must* collapse to one cache key, otherwise the KV entry fragments into one record per surface form (one for the lower-prefix paste, one for the whitespaced paste, one for the URL-embedded paste…) and the download filename changes between visits, breaking any automation that ingests by name. Pinned by asserting `.toEqual()` between the canonical `REAL_ID` and four equivalent forms (lower-prefix `chij…`, whitespaced `\t  …\n`, embedded in a long Google Maps URL, plus separate aliasing across the MOCK_ family (upper/lower/mixed) and the 0x family (upper-hex vs lower-hex). **Second: canonical-form idempotency.** `normalisePlaceId(normalisePlaceId(x).raw)` must equal `normalisePlaceId(x)` — the cache writes a key once and reads it many times; if the canonical `raw` weren't itself a fixed point, a normalise → store → re-normalise → fetch loop would silently miss its own writes. Started from a non-canonical input (lower-prefix `chij…`) so the second pass actually exercises the canonicalisation arrow rather than passing vacuously, then extended over MOCK_ and 0x families. **Third: PLACE_ID_PATTERN minimum-length floor.** `ChIJ[\w-]{20,}` and `0x[0-9a-fA-F]+:0x[0-9a-fA-F]+` and `MOCK_[A-Z0-9_]+` carry per-family length floors; a refactor that loosened `{20,}` to `{1,}` or replaced `+` with `*` would let almost-real stubs through and the live SF call would 404 with no validation signal at our edge — pin the floor at the boundary so the regression reads as a red test, not a confusing upstream 404.
